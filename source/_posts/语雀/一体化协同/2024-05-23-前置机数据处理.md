## 注意事项
1、查询数据时使用**RowGuid**查询，避免返回多条结果，插入多条结果
2、修改数据时，需要在up_data_reconciliation_v4插入一条记账信息
3、需要使用cd_batch在up_data_reconciliation_v4中查询最新一个cd_batch，然后cd_batch+1用户需要插入的数据编号
## 错误数据查询SQL
```sql
SELECT
<!--more-->
	id,
	cd_batch,
	error_type,
	table_name,
	projectno,
	cd_time,
	jl_sjly,
	jl_lyip,
	jl_bmmc,
	jl_xzqhdm,
	jl_zzjgdm,
	count_time,
	jl_gxsj,
	systemname,
	localtaskcode,
	taskhandleitem,
	FRONT_TIME 
FROM
	up_pro_quality_v4_error 
WHERE
	jl_sjly = '前置机' 
	AND NOT (
		error_type IN ( ";01", ";0101" ) 
AND projectno IN ( SELECT projectno FROM up_pro_accept_v4 )) 
ORDER BY
	table_name,
	error_type;

select * from sharedb.up_data_reconciliation_v4 where cd_batch like '11441823MB2D123064%' order by cd_time desc,cd_batch desc;
```

关注字段id（**id-->RowGuid**）、table_name、error_type

- id
- cd_batch
- error_type
- table_name

### 4）修改处理模板sql的数据
主要有**rowguid 、Cd_Batch、其他**

- **rowguid 值为**《错误数据查询SQL》中的id
- **Cd_Batch 值：**
```
如：
cd_batch=11441823MB2D1230642021122200018
a)18位代码11441823MB2D123064+日期20211222+编号00018组成
b)获取《错误数据查询SQL》中的Cd_Batch 
c)根据表up_data_reconciliation_v4 查询当前日期最新的一次编号
select * from sharedb.up_data_reconciliation_v4 where cd_batch like '11441823MB2D123064%' order by cd_time desc,cd_batch desc;
d)编号18位代码11441823MB2D123064+今天日期（年月日，如20211224）+已有编号+1（如最新的为12，这编号13
如查询up_pro_accept_v4的11441823MB2D123064在2021年12月22日的最新编码
select * from sharedb.up_data_reconciliation_v4 where cd_batch like '11441823MB2D123064%' and table_name='up_pro_accept_v4'  order by cd_batch desc,cd_time desc;
获取结果如下：
15959	up_pro_process_v4	11441823MB2D1230642021122200018	1	2021-12-22 18:20:18
15960	up_pro_result_v4	11441823MB2D1230642021122200018	1	2021-12-22 18:20:18
15958	up_pro_result_v4	11441823MB2D1230642021122200017	1	2021-12-22 18:20:18
这cd_batch为11441823MB2D1230642021122200019
```

- **Cd_Operation(IDU,增删改)：正常有三类IDU,分别代表了增删改**
### 5）根据错误代码调整数据逻辑
根据error_type的错误状态码，参考《国家政务服务平台事项和办件数据质检规则v4.1-20210416》，修改数据逻辑，比如1402，错误原因：受理时间大于等于申请时间；1402。
调整受理时间大于申请时间，在申请时间上加1分钟
> **ApplyDate:2021-12-22 12:23:56**
> **AcceptDate:2021-12-22 12:21:24**

调整AcceptDate为2021-12-22 12:25:56
