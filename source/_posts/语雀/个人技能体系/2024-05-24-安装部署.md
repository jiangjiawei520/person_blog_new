# 简介
## 1.1 简介
MongoDB 是由C++语言编写的，是一个基于分布式文件存储的开源数据库系统。在高负载的情况下，添加更多的节点，可以保证服务器性能。
<!--more-->
MongoDB 旨在为WEB应用提供可扩展的高性能数据存储解决方案。
MongoDB 将数据存储为一个文档，数据结构由键值(key=>value)对组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组。
官网： [https://www.mongodb.com](https://www.mongodb.com/)

## 1.2 安装MongoDB单机测试

### 1.2.1 目录规划
```bash
mkdir /usr/local/opt    #临时目录，用于上传压缩包
cd /usr/local/  #mongodb的安装目录
```

### 1.2.2 上传MongoDB安装包
```bash
cd  /usr/local/opt/   # 跳转到临时目录
#wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-3.6.8.tgz
rz   #通过rz 上传mongodb-linux-x86_64-3.6.8.tgz
```

### 1.2.3 安装
1、解压
```bash
tar -zxvf  mongodb-linux-x86_64-3.6.8.tgz   # 解压
mv mongodb-linux-x86_64-3.6.8  mongodb  #重命名
mv /usr/local/opt/mongodb  /usr/local/  # 将解压包拷贝到指定目录
```
2、MongoDB 的可执行文件位于 bin 目录下，所以可以将其添加到 PATH 路径中：
```bash
vim /etc/profile

 #加入以下内容
export MONGODB_HOME=/usr/local/mongodb
export PATH=$MONGODB_HOME/bin:$PATH

 #保持退出后，执行以下命令，立即生效
source /etc/profile
```

### 1.2.4 创建数据库目录

- 创建数据库的目录

MongoDB的默认数据存储在跟目录data/db目录下，但是这个目录在安装过程不会自动创建，所以你需要手动创建data目录，并在data目录。
注意： data/db 是 MongoDB 默认的启动的数据库路径。如果你的数据库目录不是/data/db，可以通过 --dbpath 来指定。(mongod --dbpath /usr/local/mongo/data)
如果是不是分片，则可以暂时不需要创建数据库的目录。后续在分配中创建。
```bash
mkdir  -p  /usr/local/mongodb/data  # 创建数据库数据目录

mkdir  -p  /usr/local/mongodb/logs  # 创建数据库日志目录
```

### 1.2.5 启动
先测试一下 MongoDB 是否安装成功：
```bash
[root@srv15209 mongodb]# pwd
/usr/local/mongodb
```
显示 MongoDB 的bin/mongod 路径则表示安装成功。

- 启动mongodb

你可以再命令行中执行mongo安装目录中的bin目录执行mongod命令来启动mongdb服务。
MongoDB后台运行，启动时只需添加 --fork函数即可。可以在日志路径后面添加--logappend，防止日志被删除。
```bash
[root@srv15209 mongodb]# mongod --fork --dbpath=/usr/local/mongodb/data --logpath=/usr/local/mongodb/logs/mongodb2.log --logappend
about to fork child process, waiting until server is ready for connections.
forked process: 14440
child process started successfully, parent exiting
```
--logpath= 后不能有空格
成功：

```bash
forked process: 14440
child process started successfully, parent exiting
```

### 1.2.6 远程连接
MongoDB数据库启动后，默认只能在本地使用，如果远程访问，需要进行一下设置。
1、关闭防火墙，或开放对应的端口
```bash
systemctl stop firewalld 
```
2、启动命令中添加 。--bind_ip=0.0.0.0
```bash
[root@srv15209 mongodb]# mongod --fork --dbpath=/usr/local/mongodb/data --logpath=/usr/local/mongodb/logs/mongodb2.log --logappend --bind_ip=0.0.0.0
about to fork child process, waiting until server is ready for connections.
forked process: 14770
child process started successfully, parent exiting
```

## 1.3 centos7搭建mongodb副本集

### 1.3.1 安装mongo4.2.0
下载Mongo:mongodb-linux-x86_64-3.6.8.tgz
将压缩文件mongodb-linux-x86_64-3.6.8.tgz拷贝到：/usr/local/opt
```bash
mkdir /usr/local/opt/
cd /usr/local/opt/
wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-3.6.8.tgz
tar -zxvf  mongodb-linux-x86_64-3.6.8.tgz   # 解压
mv mongodb-linux-x86_64-3.6.8  mongodb  #重命名
mv /usr/local/opt/mongodb  /usr/local/  # 将解压包拷贝到指定目录
cd /usr/local/mongodb
mkdir -p /usr/local/mongodb/node{1,2,3}
mkdir -p /usr/local/mongodb/log/log{1,2,3}
mkdir -p /usr/local/mongodb/conf/
touch /usr/local/mongodb/conf/node{1,2,3}.conf
```

### 1.3.3 启动方法
先按下面配置后启动，使用配置文件方式，详细见配置node1.conf node2.conf node3.conf
```bash
# chmod -R 740 /usr/local/mongodb
cd /usr/local/mongodb/
yum -y install nano
nano /usr/local/mongodb/conf/node1.conf
nano /usr/local/mongodb/conf/node2.conf
nano /usr/local/mongodb/conf/node3.conf

./bin/mongod -f /usr/local/mongodb/conf/node1.conf
./bin/mongod -f /usr/local/mongodb/conf/node2.conf
./bin/mongod -f /usr/local/mongodb/conf/node3.conf
```
node1.conf:：
```bash
#内容
#指定数据库路径
dbpath =/usr/local/mongodb/node1
#指定MongoDB日志文件，注意是指定文件不是目录		
logpath =/usr/local/mongodb/log/log1/config.log
# PID File 的完整路径，如果没有设置，则没有PID文件
#pidfilepath=/var/run/mongodb/mongod1.pid
# 集群的私钥的完整路径，只对于Replica Set 架构有效
#auth = true
keyFile=/usr/local/mongodb/key
# 设置每个数据库将被保存在一个单独的目录
#directoryperdb=true
#启用日志选项，MongoDB的数据操作将会写入到journal文件夹的文件里
#journal=true
#最大同时连接数默认2000
#maxConns=2000
#以追加的方式记录日志
logappend = true
#设置副本集名称
replSet=node
#绑定服务IP，若绑定127.0.0.1，则只能本机访问，不指定默认本地所有IP,否则可能无法通过ip+端口方式启动
bind_ip = 0.0.0.0
#端口
port = 10001
#是否后台运行,以守护进程的方式运行MongoDB，创建服务器进程
fork = true
```

# 分片部署
## 1.分片集群介绍与原理
mongodb分片

- 一般用得比较少，需要较多的服务器，还有三种的角色。
- 一般把mongodb的副本集应用得好就足够用了，可搭建多套mongodb复本集

mongodb分片技术。

- mongodb副本集可以解决数据备份、读性能的问题，但由于mongodb副本集是每份数据都一模一样的，无法解决数据量过大问题。
- mongodb分片技术能够把数据分成两份存储，假如bfd.user（bfd库，user是表）里面有100w条数据，分片能够实现50万左右存储在data1，50万左右存储在data2。
- data1、data2需要使用副本集的形式，预防数据丢失。

mongodb分片集群三种角色介绍：

|  router角色 （mongos server） | mongodb的路由，提供入口，使得分片集群对外透明。router不存储数据 |
| --- | --- |
| configsvr角色 （config server） | mongodb的配置角色，存储元数据信息。分片集群后端有多份存储，读取数据该去哪个存储上读取，依赖于配置角色。配置角色建议使用副本集 |
| shardsvr角色（shard server） | mongodb的存储角色，存储真正的数据，建议使用副本集 |

**（配置config server副本集）**mongodb3.4以后要求配置服务器也创建副本集，不然集群搭建不成功
**（配置分片副本集）**分片集群的数据角色里面存储着真正的数据，所以数据角色一定得使用副本集。
**（配置路由服务器mongos(路由角色)）**mongodb中的router（mongos）角色只负责提供一个入口，不存储任何的数据，router最重要的配置是指定configsvr的地址，使用副本集ip+端口的方式指定配置多个router，任何一个都能正常的获取数据。
**（启用分片）**搭建了mongodb配置服务器、路由服务器，各个分片服务器，不过应用程序连接到mongos路由服务器并不能使用分片机制，还需要在程序里设置分片配置，让分片生效。

**Router负责接受访问，然后去config服务器中查询元数据，将数据存储信息返回给Router，Router服务器根据元数据的存储信息在分片服务器上读取或者写入数据。**

### 1.2 部署设计
#### 1.2.1 机器规划
为了更好的均衡机器的压力，把每个机器既可以作为主节点、副本节点、仲裁节点，这样压力就会均衡很多了。

- **依赖关系：**

    当用户通过router（mongos路由）角色插入数据时，需要从configsvr知道这份数据插入到哪个节点， 然后执行插入动作插入数据到sharedsvr
    当用户通过router（mongos路由）角色获取数据时，需要从configsvr知道这份数据是存储在哪个节点，然后再去sharedsvr获取数据

- **服务器规划：**
| 173.16.122.124 | 173.16.122.174 | 173.16.122.236 |
| --- | --- | --- |
| mongos | mongos | mongos |
| config server | config server | config server |
| shard1 主节点 | shard1 副节点 | shard1 仲裁 |
| shard2 仲裁 | shard2 主节点 | shard2 副节点 |
| shard3 副节点 | shard3 仲裁 | shard3 主节点 |

#### 1.2.2 端口规划
由于一个机器需要同时部署 mongos、config server 、shard1、shard2、shard3，所以需要用端口进行区分。

> shard1：27001

#### 3.6.4 启动分片
目前配置服务、路由服务、分片服务、副本集服务都已经串联起来了，但我们的目的是希望插入数据，数据能够自动分片。连接在mongos上，准备让指定的数据库、指定的集合分片生效。
指定数据库里需要分片的集合和片键:
我们设置filewarehouse的 GeoFile.chunks 表需要分片，根据 id 自动分片到 shard1，shard2，shard3，.. 上面去。
要这样设置是因为不是所有mongodb 的数据库和表 都需要分片！!
```bash
1、启用分片功能

use admin

2、对集合启用分片功能
sh.shardCollection("filewarehouse.GeoFile.chunks", {"files_id":1})
sh.status()
```
