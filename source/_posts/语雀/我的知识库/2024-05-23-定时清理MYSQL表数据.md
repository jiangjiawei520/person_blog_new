        使用mysql 自带的事件,定时任务来定期处理这些数据。
## 1.打开MYSQL ,查看你的定时任务开关是否开启
> SHOW VARIABLES LIKE 'event_scheduler'
> 
> 如果value是off 需要你打开一下
> SET GLOBAL event_scheduler = ON

## 2.写定时任务的事件
> DELIMITER $$
> DROP EVENT IF EXISTS deleteLog;
> CREATE EVENT deleteLog
> ON SCHEDULE EVERY 300 SECOND
> ON COMPLETION PRESERVE
> DO BEGIN
> delete from sys_log where TO_DAYS(now())-TO_DAYS(log_time)>90;
> END$$
> DELIMITER ；

详细说明:
SCHEDULE EVERY 300 SECOND : 每5分钟执行一次(YEAR，MONTH，DAY，HOUR，MINUTE 或者SECOND)
sys_log  : 你要操作的表
log_time : 表中含有事件的字段(必须有含有时间的字段 不然没有办法对比)
TO_DAYS(now())-TO_DAYS(log_time)>90 : 当前时间减去字段(log_time) 时间 大于 90 就 删除这些数据

## 4. 测试


```

select * from mysql.event;

#
SHOW VARIABLES LIKE 'event_scheduler'

#
SET GLOBAL event_scheduler = ON

SELECT TO_DAYS(now())-TO_DAYS(call_time), call_time from proxy_log_visit
SELECT TO_DAYS(now())-TO_DAYS(call_time), call_time from sc_log_visitinfo


#
DELIMITER $$
DROP EVENT IF EXISTS deleteLog;
CREATE EVENT deleteLog
ON SCHEDULE EVERY 24 HOUR
ON COMPLETION PRESERVE
DO BEGIN
delete from proxy_log_visit where TO_DAYS(now())-TO_DAYS(call_time)>5;
delete from sc_log_visitinfo where TO_DAYS(now())-TO_DAYS(CALL_TIME)>5;
END$$
DELIMITER ；


#
select * from mysql.event;
```
