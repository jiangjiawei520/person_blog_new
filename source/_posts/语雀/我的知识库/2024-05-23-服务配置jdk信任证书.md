主要问题对接时，我们的应用服务请求客户的接口报错 The connection observed an errorio.netty.handler.codec.DecoderException: javax.net.ssl.SSLHandshakeException: General SSLEngine problem，但在本地开发环境 Windows 10 系统中直接请求这个 https 接口是正常的。

### 原因分析
根据服务异常信息，可以大略猜到是 SSL 证书的问题。
查看客户的证书信息，发现证书机构是 CFCA（中国金融认证中心），并不是常见的 GlobalSign、DigiCert、GeoTrust 等证书机构。
![image.png](https://cdn.nlark.com/yuque/0/2022/png/12484160/1652165826321-41427171-f2a1-4fdc-a93b-31d5a4efd4e7.png#averageHue=%23f4f3f3&clientId=u87634209-b53f-4&from=paste&id=ud8ab0c81&originHeight=651&originWidth=1200&originalType=url&ratio=1&rotation=0&showTitle=false&size=258379&status=done&style=none&taskId=ubd00abcd-e39e-4fd6-afcd-769bd3d18bc&title=)
<!--more-->
在服务器上使用命令 keytool -list -v -keystore $JAVA_HOME/jre/lib/security/cacerts （默认密码为 changeit ）查看 JDK 的根证书机构库，没有发现 CFCA 这个机构。
![image.png](https://cdn.nlark.com/yuque/0/2022/png/12484160/1652165829815-c6d94d9e-48c3-4d91-b937-17138324a3ff.png#averageHue=%234d4637&clientId=u87634209-b53f-4&from=paste&id=u4d525914&originHeight=62&originWidth=752&originalType=url&ratio=1&rotation=0&showTitle=false&size=45514&status=done&style=none&taskId=ueebe152a-b087-4579-9f2e-f12ce8c0973&title=)
**因为根证书机构 CFCA 不被信任，我们的 Java 服务就无法判断此 https 站点证书的合法性了，所以就报错了。**
在 Windows 10 系统中打开证书管理工具（通过快捷键 Win + R 打开 运行窗口，再输入 certlm.msc 后回车） ，是可以看到 CFCA 这个机构的根证书的，所以在本地开发环境 Windows 10 系统中直接请求这个 https 接口是正常的。
![image.png](https://cdn.nlark.com/yuque/0/2022/png/12484160/1652165834763-f67ee5ee-0698-43f2-b703-5e6dcbf23e3e.png#averageHue=%23dad7cf&clientId=u87634209-b53f-4&from=paste&id=ud245b891&originHeight=393&originWidth=1200&originalType=url&ratio=1&rotation=0&showTitle=false&size=548093&status=done&style=none&taskId=ucd58c789-1dfa-4850-a09a-ce00bbae577&title=)

### 解决办法
**把 CFCA 这个机构的根证书导入 JDK 信任库。**

1. 导出 CFCA 根证书为 cfca.cer 。

// -keystore：指定 JDK 证书库文件
// -file：指定要导入的证书

# keytool -import -alias cfca -keystore ${JAVA_HOME}/jre/lib/security/cacerts -file ${JAVA_HOME}/jre/lib/security/cfca.cer
