# **nginx的配置文件结构**
```
...              #全局块
events {         #events块
...
}
http      #http块
<!--more-->
{
...   #http全局块
upstream
{
…
}
server        #server块
{
...       #server全局块
location [PATTERN]   #location块
{
...
}
location [PATTERN]
{
...
}
}
server
{
...
}
}

1、全局块：配置影响nginx全局的指令。一般有运行nginx服务器的用户组，nginx进程pid存放路径，日志存放路径，配置文件引入，允许生成worker process数等。
2、events块：配置影响nginx服务器或与用户的网络连接。有每个进程的最大连接数，选取哪种事件驱动模型处理连接请求，是否允许同时接受多个网路连接，开启多个网络连接序列化等。
3、http块：可以嵌套多个server，配置代理，缓存，日志定义等绝大多数功能和第三方模块的配置。如文件引入，mime-type定义，日志自定义，是否使用sendfile传输文件，连接超时时间，单连接请求数等。
4、server块：配置虚拟主机的相关参数，一个http中可以有多个server。
5、location块：配置请求的路由，以及各种页面的处理情况。
**几个常见配置项：**
1.$remote_addr 与 $http_x_forwarded_for 用以记录客户端的ip地址；
2.$remote_user ：用来记录客户端用户名称；
3.$time_local ： 用来记录访问时间与时区；
4.$request ： 用来记录请求的url与http协议；
5.$status ： 用来记录请求状态；成功是200；
6.$body_bytes_s ent ：记录发送给客户端文件主体内容大小；
7.$http_referer ：用来记录从那个页面链接访问过来的；
8.$http_user_agent ：记录客户端浏览器的相关信息；
# **HTTPS配置**
## **生成ssl证书**
### **下载安装openssl**
[http://slproweb.com/products/Win32OpenSSL.html](http://slproweb.com/products/Win32OpenSSL.html)
注意不要下载Light版。
下载后安装并设置环境变量OPENSSL_HOME=openssl安装目录下bin目录；OPENSSL_CONF=openssl安装目录下bin目录下openssl.cfg。
注意：OPENSSL_CONF需指明openssl.cfg的完整路径。
### **生成ssl证书**
1. 创建证书路径
在nginx目录下创建cert文件夹，用于存放证书。
2. 创建私钥
进入cert文件夹，windows系统在路径栏输入cmd打开命令窗口，Linux系统在当前路径下执行命令。genrsa表示生成rsa私钥，des3表示加密方式，out指定输出的文件，1024表示私钥长度。
openssl genrsa -des3 -out SSL.key 1024
3. 创建 csr 证书
openssl req -new -key SSL.key -out SSL.csr
4. 删除密码
复制SSL.key并重命名为SSL.key.org
执行命令
openssl rsa -in shidian.key.org -out shidian.key
5. 生成CRT证书
openssl x509 -req -days 365 -in SSL.csr -signkey SSL.key -out SSL.crt
## **修改nginx配置**
修改server模块的配置
server {
# https监听的端口，注意后面必须有ssl标识。
listen       8088 ssl;
# 域名，可不配置；在多域名时，可按域名匹配请求分发规则。
server_name  localhost;
#ssl的crt证书完整路径。
ssl_certificate      D://nginx-1.12.2//cert//SSL.crt;
#ssl的私钥完整路径
ssl_certificate_key  D://nginx-1.12.2//cert//SSL.key;
#ssl的session缓存策略；off禁用缓存，session不重用；none禁用缓存，session可重用；builtlin：1000使用openssl缓存，仅由一个工作进程使用，1000表示可缓存1000个session；shared：SSL:1m，所有进程可共享缓存，1m表示缓存空间大小，1m约4000个session。
ssl_session_cache    shared:SSL:1m;
#session缓存有效时间，5m表示5分钟。
ssl_session_timeout  5m;
#ssl加密套件,设置使用openssl的加密套件，HIGH表示大于128位的加密算法，MEDIUM表示128位加密算法，LOW表示64位和56位的加密算法；通过：分隔算法列表；!表示移除，不再添加；-表示移除，后续列表可再次添加；+表示移至末尾；。
ssl_ciphers  HIGH:!aNULL:!MD5;
#是否由服务器决定采用哪种加密算法；设为on时，配合ssl_ciphers使用，设为off时，需ssl版本支持tlsv1.2 tlsv1.3。
ssl_prefer_server_ciphers  on;
location / {
} 
}

修改配置后重启nginx。
注：必须先关闭nginx再启动。reload和reopen均不生效。
