1、明确表空间名称，及其涉及到的用户，数据库ip及其端口/实例名
2、备份导出数据库
　　一个数据库是由多个表空间（tablespace）组成，一个表空间又由多个段（segment）组成，一个段又由多个区（extent）组成，一个区则由多个块（block）组成。
对于我们在数据库里新建数据库（database），在数据库中建立多个表空间（tablespace），在每个表空间内建表。例如我们可以分配多个用户，在user1用户下建立table1，table2,user2下建立table3，table4，user1、user2就等于不同的表空间，table1、table2是建立在表空间下的不同段（segment），而每张表的每个数据就是块（block），一列数据可看做一个区（extent），区满了之后不断扩展就组成了表。
oracle11g的新特性，数据条数是0时不分配segment，所以就不能被导出
(1).用sys用户查指定用户下有多少张空表
select count(*) from dba_tables a where a.num_rows = 0 and owner='JSYDEOS';
(2).分别用相应用户身份登录，使用拼接SQL语句，得到更改空表属性状态语句，再执行.
<!--more-->
select 'alter table '||table_name||' allocate extent;' from user_tables where num_rows=0

CREATE BIGFILE TABLESPACE TDGLYWK DATAFILE 
'F:\app\Administrator\oradata\orcl\TDGLYWK.dbf' SIZE 100M AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED LOGGING EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO;
4、分别创建所有用户
/*打开PLSQL，执行如下语句，创建jsydeos用户 */
create user jsydeos identified by jsydeos default tablespace TDGLYWK temporary tablespace temp;
/*授权*/
/*with admin option的意思是被授予该权限的用户有权将某个权限(如create any table)授予其他用户或角色，取消是不级联的*/
  grant connect ,resource,dba to JSYDEOS with admin option;
  GRANT CREATE ANY VIEW TO "JSYDEOS";
  GRANT SELECT ANY TABLE TO "JSYDEOS";
  GRANT UNLIMITED TABLESPACE TO "JSYDEOS" WITH ADMIN OPTION;
/* 修改密码有效期 */
ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;

/*打开PLSQL，执行如下语句， 创建jsydbps用户 */
create user jsydbps identified by jsydbps default tablespace TDGLYWK temporary tablespace temp;
/*授权*/
  grant connect ,resource,dba to jsydbps with admin option;
/* 修改密码有效期 */
ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;

备注：
重新做全量导入时需先删除对应用户，再重新创建用户后进行用户下所有表的导入，删除用户的语句如下：
drop user jsydeos cascade;
drop user jsydbps cascade;
drop user jsydtxk cascade;

5、还原数据库
方式一：打开cmd窗口，执行imp导入sql语句，还原数据库，其中“file”为备份数据库dmp文件存放路径。
分对应用户进行还原：
（1）还原jsydbps用户下的数据：
数据库连接信息根据真实情况进行修改，此处以现场数据库信息进行说明
imp 'jsydbps/jsydbps@172.16.101.15:1521/orcl' buffer=65536 file=D:\GDJSYDSP \fulldatabase_20171022_bps.dmp log=C:\GDJSYDSP\imp_fulldatabase_20171022_jsydbps.log full=y
方式二：plsql导入，切换到对应的用户导入对应的.dmp文件

--查看表空间下的所有表
SELECT * FROM User_Tables  WHERE TABLESPACE_NAME='TDGLYWK';
--查看用户信息
SELECT * FROM DBA_USERS A WHERE A.USERNAME='TDGLEOS';

--删除oracle表空间下的所有表
drop tablespace TDGLYWK including contents;

--修改表空间名字
alter tablespace TDGLYWK rename to TDGLYWK1;

--修改用户名字
select username from dba_users order by created;
update user$ set name='TDGLEOS'WHERE NAME='JSYDEOS';
update user$ set name='TDGLBPS'WHERE NAME='JSYDBPS';
alter system checkpoint;
alter system flush shared_pool;
ALTER USER TDGLEOS IDENTIFIED BY TDGLEOS;
ALTER USER TDGLBPS IDENTIFIED BY TDGLBPS;
commit;
--修改用户密码
alter user TDGLEOS identified by TDGLEOS;
alter user TDGLBPS identified by TDGLBPS;

--修改用户的表空间
alter table TDGLEOS move tablespace TDGLYWK


1.1.1 现场数据库的备份还原操作详细说明（实例）
现场数据库使用的表空间统一为”GDJSYD”，涉及的用户信息一共有四个，分别为：“jsydeos、jsydbps、jsydtxk、jsydjh”，数据库的ip：端口/实例名为：172.16.101.15:1521/orcl
 
1.1.1.1 备份数据库
1.针对oracle11版本在进行默认备份时，忽略空表的备份，而后续会需要再用到相应表单会出现因空表未还原而报错现象，因此在备份前，需要更改空表属性，使得空表可以被备份。
（1）/*用sys用户查指定用户下有多少张空表*/
select count(*) from dba_tables a where a.num_rows = 0 and owner='JSYDEOS';
select count(*) from dba_tables a where a.num_rows = 0 and owner='JSYDBPS';
select count(*) from dba_tables a where a.num_rows = 0 and owner='JSYDTXK';
select count(*) from dba_tables a where a.num_rows = 0 and owner='JSYDJH';
（2）/*分别用相应用户身份登录，使用拼接SQL语句，得到更改空表属性状态语句，再执行即可*/
select 'alter table '||table_name||' allocate extent;' from user_tables where num_rows=0
得到的更改空表属性语句举例如下：
alter table OA_BYSL allocate extent;
alter table OA_CB allocate extent;
